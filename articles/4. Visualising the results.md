# 4. Visualising the results

### Timing posteriors

The inferred timing posteriors can be visualised on the patient’s
clinical timeline in two complementary ways: as histograms overlaid on a
single timeline, or as stacked densities. In both plots, treatment
periods are shown as coloured rectangles and sampling dates as points on
the timeline.

``` r

TOSCA::plot_timing(UPN06)
```

![](4.%20Visualising%20the%20results_files/figure-html/unnamed-chunk-3-1.png)

``` r

TOSCA::plot_timing_MAP(UPN06)
```

![](4.%20Visualising%20the%20results_files/figure-html/unnamed-chunk-4-1.png)

By default the x-axis is expressed in calendar dates. If preferred,
timings can be displayed in days relative to a clinically meaningful
reference event. For UPN06, we use the allo-HSCT date as the origin:

``` r

allo_HSCT <- "2014-04-23"
TOSCA::plot_timing_days(UPN06, days_from = allo_HSCT, event_name = "allo-HSCT")
```

![](4.%20Visualising%20the%20results_files/figure-html/unnamed-chunk-5-1.png)

Summary statistics can likewise be computed relative to a reference date
using
[`days_from()`](https://caravagnalab.github.io/TOSCA/reference/days_from.md).
This is particularly useful for classifying the timed event as *de novo*
(originating after the reference) or *pre-existing* (originating before
it). For CNA-mediated resistance, the parameter `t_cna` and the length
of the corresponding CNA, as specified in the mutations input, must be
provided:

``` r

TOSCA::days_from(UPN06, from = allo_HSCT, parameter = "t_cna", cna_length = 47911693)
#> # A tibble: 1 × 4
#>   median  mean    q5      q95     
#>   <drtn>  <drtn>  <drtn>  <drtn>  
#> 1 95 days 96 days 65 days 132 days
```

Consistent with the findings reported in the reference paper, the
posterior places the CNLOH of the HLA locus after the allo-HSCT and in
close temporal proximity to ganciclovir exposure, supporting a *de novo*
origin of the resistance event.

### Dynamic parameters

To assess the inferred dynamic parameters,
[`plot_prior_vs_posterior()`](https://caravagnalab.github.io/TOSCA/reference/plot_prior_vs_posterior.md)
displays the prior density alongside the posterior histogram for each
parameter. The posterior median is indicated by a vertical red line.

``` r

TOSCA::plot_prior_vs_posterior(UPN06)
```

![](4.%20Visualising%20the%20results_files/figure-html/unnamed-chunk-7-1.png)

### Posterior predictive check

To verify that the inferred parameters are compatible with the observed
data,
[`plot_ppc()`](https://caravagnalab.github.io/TOSCA/reference/plot_ppc.md)
displays the multivariate posterior predictive distribution (PPD) for
each mutation group. The observed mutation count is shown as a red
point; the PPD is represented as a cloud of draws, colour-coded by their
distance from the observed value.

``` r

TOSCA::plot_ppc(UPN06)
```

![](4.%20Visualising%20the%20results_files/figure-html/unnamed-chunk-8-1.png)

## MCMC diagnostics

TOSCA exposes a suite of diagnostic plots built on the
[bayesplot](https://mc-stan.org/bayesplot/) package. These should be
inspected routinely to verify that the sampler has explored the
posterior adequately. Available diagnostics include chain trace plots,
$`\hat{R}`$ plots, effective sample size (ESS) plots, and divergent
transitions.

``` r

TOSCA::plot_mcmc_chains(UPN06)
```

``` r

TOSCA::rhats_plot(UPN06)
```

``` r

TOSCA::ess_plot(UPN06)
```

``` r

TOSCA::plot_divergent_transitions(UPN06)
```
