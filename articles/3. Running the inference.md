# 3. Running the inference

## Running the inference

Once all inputs are in place and the TOSCA object has been correctly
initialised, the inference is launched with a single call to
[`fit()`](https://caravagnalab.github.io/TOSCA/reference/fit.md). The
function takes the TOSCA object as input and populates its inference
results slot. For resistances mediated by a CNA event, as is the case
for UPN06, `model_name` should be set to `"CNA"`. The `dormancy`
argument should be set to `TRUE` when the patient has received a
chemotherapeutic agent known to induce tumour cell dormancy, such as
cytarabine. Additional arguments allow fine-grained control over the
MCMC sampler (number of chains, iterations per chain, warmup length, and
so on); see
[`?TOSCA::fit`](https://caravagnalab.github.io/TOSCA/reference/fit.md)
for full details.

``` r

UPN06 <- TOSCA::fit(
  UPN06,
  model_name = "CNA",
  dormancy   = TRUE
)
```

Upon completion,
[`fit()`](https://caravagnalab.github.io/TOSCA/reference/fit.md) prints
a summary report containing:

- **Convergence diagnostics:** whether all $`\hat{R} < 1.01`$, the
  number of divergent transitions, and whether the E-BFMI fell below 0.3
  in any chain.
- **Preliminary posterior predictive check:** whether the observed
  mutation count for each group falls between the mode and the 90th
  percentile of the posterior predictive distribution.
- **Parameter summary:** posterior estimates for each inferred
  parameter, including the posterior predictives.
